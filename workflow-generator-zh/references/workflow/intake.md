# 摄入提示

用于锁定用户输入的工作流思路。

硬约束：

- 第一次回复时，先收思路，不开始生成工作流。
- 若用户描述过于模糊（缺少领域、输入、输出中的任意两项），必须追问，不得直接猜测。
- 输入优先级默认是：用户明确要求 > 领域最佳实践 > 通用模板。
- 用户的一句话思路描述是起点，不是完整规格。

必须确认：

1. 这个工作流要解决什么问题（一句话）
2. 输入材料是什么（文件、数据、配置……）
3. 期望输出什么（报告、文档、代码、配置……）
4. 有哪些硬性约束（格式要求、质量标准、工具限制……）
5. 是否需要特定的输出格式（.xlsx、.docx、.json、.html……）

首次回复时，必须明确告诉用户可以直接发送本地路径，示例：

- `D:\需求文档.docx`
- `D:\测试用例.xlsx`
- `D:\设计文档.docx`

如果用户一次没给全，不要连续抛很多问题。优先确认：

1. 工作流要解决什么问题
2. 输入材料是什么
3. 期望输出什么
4. 有什么硬性约束

信息不全时，不要只说"请补充信息"。必须告诉用户：

1. 缺失信息影响工作流的哪些部分
2. 当前是否可以有限继续
3. 有限继续会牺牲什么
4. 用户可以选择补充、确认没有、或暂缓
5. 推荐路径是什么
